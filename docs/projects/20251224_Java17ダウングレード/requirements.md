## 2. 背景・目的

### 2.1 背景

- 現行システムは Java 21 で構築されている
- 運用環境や開発環境の制約により Java 17 への統一が必要

### 2.2 目的

- Java バージョンを 21 から 17 にダウングレードする
- 関連するドキュメントを更新する

---

## 3. スコープ

### 3.1 対象範囲

| 対象 | 変更内容 |
|------|----------|
| pom.xml | java.version を 21 → 17 に変更 |
| README.md | Java バージョン記載を更新 |
| docs/specs/architecture.md | 技術スタック情報を更新 |

### 3.2 対象外

- コードの変更（Java 21 固有機能は使用していないため不要）
- 依存ライブラリのバージョン変更（Spring Boot 3.2 は Java 17 をサポート）

---

## 4. 受入条件

| # | 条件 | 確認方法 |
|---|------|----------|
| 1 | pom.xml の java.version が 17 になっている | ファイル確認 |
| 2 | アプリケーションがビルドできる | mvnw clean compile |
| 3 | 既存テストがパスする | mvnw test |
| 4 | ドキュメントが更新されている | README.md, architecture.md 確認 |

---

## 5. 技術的考慮事項

### 5.1 互換性

| 項目 | 状況 |
|------|------|
| Spring Boot 3.2 | Java 17+ をサポート（互換性あり） |
| MyBatis 3.0.3 | Java 17 をサポート（互換性あり） |
| Java 21 固有機能 | 使用していない（影響なし） |

### 5.2 リスク

- なし（Java 21 固有の機能を使用していないため）
