## 1. 概要

### 1.1 本設計書の目的

本案件におけるバックエンドの変更有無と、その理由を明確化する。

### 1.2 バックエンドの役割

| 責務 | 本案件での変更 |
|------|---------------|
| データ管理 | **変更なし** |
| ビジネスロジック | **変更なし** |
| API提供 | **変更なし** |
| バリデーション | **変更なし** |

### 1.3 変更なしの理由

本案件はフロントエンドのみの改善であり、以下の理由によりバックエンドの変更は不要である:

1. **静的HTMLファイルの追加のみ**: `index.html`を`static/`フォルダに配置することで、Spring Bootが自動的に配信する
2. **新規API不要**: ナビゲーションカードは静的リンクであり、動的データの取得を行わない
3. **データモデル変更なし**: 既存のエンティティ（Todo, Project, User）に変更なし
4. **HelloController**: ルートパス（`/`）のマッピングを削除するが、これは機能削除であり新規開発ではない

## 7. 実装変更箇所

### 7.1 変更ファイル

| ファイルパス | 変更内容 | 理由 |
|--------------|----------|------|
| HelloController.java | `@GetMapping("/")`の削除 | 静的ファイル`index.html`との競合回避 |

### 7.2 変更詳細

#### HelloController.java

**変更前**:
```java
@RestController
public class HelloController {
    @GetMapping("/")
    public String hello() {
        return "こんにちは！Spring Bootアプリケーションへようこそ！...";
    }
}
```

**変更後**:
```java
@RestController
public class HelloController {
    // ルートパスのマッピングを削除
    // static/index.html が自動的に配信される
}
```

または、クラスごと削除（他のメソッドがない場合）。
