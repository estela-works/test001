## 1. 案件概要

### 1.1 案件の背景

個人のタスク管理において、以下の課題が存在する：
- やるべきことが頭の中にあり、忘れてしまう
- タスクの進捗状況が見えない
- 完了したタスクと未完了のタスクの区別がつかない

これらの課題を解決するため、シンプルなタスク管理Webアプリケーションを新規構築する。

### 1.2 案件の目的

Spring Bootを使用したToDoアプリケーションを構築し、以下の価値を提供する：
- タスクを可視化し、忘れを防止する
- 進捗を追跡し、達成感を得られる
- 完了状態を管理し、残タスクを明確にする

### 1.3 対象ユーザー

| ユーザー | 利用シーン |
|---------|-----------|
| 個人 | 日常のタスク管理 |
| 開発者 | 学習・デモ用途 |

※ 単一ユーザーを想定（認証なし）

## 2. 要求一覧

### 2.1 ビジネス要求

| ID | 要求 | 優先度 | 理由 |
|----|------|--------|------|
| BR-001 | タスクを登録できること | 必須 | やるべきことを記録するため |
| BR-002 | タスクを一覧で確認できること | 必須 | 登録したタスクを把握するため |
| BR-003 | タスクの完了/未完了を管理できること | 必須 | 進捗を追跡するため |
| BR-004 | 不要なタスクを削除できること | 必須 | リストを整理するため |
| BR-005 | タスクの統計情報を確認できること | 希望 | 全体状況を把握するため |
| BR-006 | 完了/未完了でフィルタできること | 希望 | 必要なタスクに集中するため |

### 2.2 ユーザー要求（ユーザーストーリー）

| ID | ユーザーストーリー | 対応BR | 優先度 |
|----|-------------------|--------|--------|
| US-001 | ユーザーとして、新しいタスクを登録したい。なぜなら、やるべきことを忘れないようにしたいから。 | BR-001 | 高 |
| US-002 | ユーザーとして、登録したタスクを一覧で見たい。なぜなら、今日やるべきことを把握したいから。 | BR-002 | 高 |
| US-003 | ユーザーとして、タスクを完了にしたい。なぜなら、達成感を得たいから。 | BR-003 | 高 |
| US-004 | ユーザーとして、不要なタスクを削除したい。なぜなら、リストを整理したいから。 | BR-004 | 高 |
| US-005 | ユーザーとして、完了/未完了で絞り込みたい。なぜなら、残りのタスクに集中したいから。 | BR-006 | 中 |
| US-006 | ユーザーとして、タスクの統計を見たい。なぜなら、全体状況を把握したいから。 | BR-005 | 中 |

## 3. スコープ定義

### 3.1 スコープ内（In Scope）

| カテゴリ | 項目 | 説明 |
|---------|------|------|
| 機能 | タスクCRUD | 作成・参照・更新・削除 |
| 機能 | 完了状態管理 | 完了/未完了のトグル |
| 機能 | 統計表示 | 総数・完了数・未完了数 |
| 機能 | フィルタリング | 完了状態での絞り込み |
| 画面 | ホーム画面 | アプリへの入口 |
| 画面 | ToDoリスト画面 | メイン操作画面 |
| 技術 | バックエンド | Spring Boot REST API |
| 技術 | フロントエンド | HTML/CSS/JavaScript |
| 技術 | データ保存 | インメモリ（ConcurrentHashMap） |

### 3.2 スコープ外（Out of Scope）

| 項目 | 見送り理由 | 将来検討 |
|------|-----------|---------|
| ユーザー認証 | MVP段階では単一ユーザー想定 | 可 |
| データ永続化（DB） | シンプルさ優先、再起動リセット許容 | 可 |
| 期限管理 | 基本機能の安定を優先 | 可 |
| 優先度設定 | 基本機能の安定を優先 | 可 |
| カテゴリ分類 | 基本機能の安定を優先 | 可 |
| 通知機能 | 基本機能の安定を優先 | 可 |
| ToDo編集（インライン） | 削除→再作成で対応可能 | 可 |
| ドラッグ&ドロップ | 基本機能の安定を優先 | 可 |
| レスポンシブ対応 | デスクトップ優先 | 可 |
| ダークモード | 基本機能の安定を優先 | 可 |
| オフライン対応 | 基本機能の安定を優先 | 可 |
