## 1. 概要

### 1.1 本設計書の目的

フロントエンドが担う画面・UIの基本設計を定義する。

### 1.2 フロントエンドの役割

| 責務 | 説明 |
|------|------|
| 画面描画 | ToDoデータをHTML要素として表示 |
| ユーザー操作 | クリック・入力イベントの処理 |
| API連携 | バックエンドAPIの呼び出し |
| 状態管理 | フィルタ状態・ToDoデータの保持 |
| フィードバック | ローディング・エラー表示 |

| 項目 | 内容 |
|------|------|
| パス | `/todos.html` |
| 目的 | ToDo管理のメイン機能を提供 |

#### レイアウト

```
┌─────────────────────────────────────────────────┐
│                   📝 ToDoリスト                  │  ← ヘッダー
├─────────────────────────────────────────────────┤
│         総数: X   完了: X   未完了: X            │  ← 統計エリア
├─────────────────────────────────────────────────┤
│  新しいToDoを追加                                │
│  [タイトル入力                              ]   │  ← 追加フォーム
│  [説明入力（任意）                          ]   │
│  [追加]                                        │
├─────────────────────────────────────────────────┤
│       [すべて] [未完了] [完了済み]               │  ← フィルタボタン
├─────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────┐ │
│  │ タイトル                                   │ │
│  │ 説明                                      │ │  ← ToDoカード
│  │ 作成日: YYYY/MM/DD                        │ │
│  │ [完了にする] [削除]                        │ │
│  └───────────────────────────────────────────┘ │
├─────────────────────────────────────────────────┤
│               ← ホームに戻る                    │  ← フッター
└─────────────────────────────────────────────────┘
```

#### 構成要素

| 要素 | 説明 | 機能 |
|------|------|------|
| ヘッダー | 画面タイトル表示 | - |
| 統計エリア | 総数/完了数/未完了数を表示 | 自動更新 |
| 追加フォーム | タイトル・説明の入力欄と追加ボタン | ToDo新規作成 |
| フィルタボタン | すべて/未完了/完了済み | 表示切替 |
| ToDoカード | ToDo情報の表示 | 完了切替・削除 |
| フッター | ホームへのリンク | 画面遷移 |

## 4. UI要素仕様

### 4.1 統計情報エリア

| 項目 | 表示形式 |
|------|----------|
| 総数 | 「総数: X」 |
| 完了 | 「完了: X」 |
| 未完了 | 「未完了: X」 |

### 4.2 追加フォーム

| 要素 | 説明 | 必須 |
|------|------|------|
| タイトル入力 | placeholder: 「タイトルを入力してください」 | 必須 |
| 説明入力 | placeholder: 「説明を入力してください（オプション）」 | 任意 |
| 追加ボタン | ラベル: 「追加」 | - |

### 4.3 フィルタボタン

| ボタン | 動作 | 初期状態 |
|--------|------|---------|
| すべて | 全件表示 | アクティブ |
| 未完了 | 未完了のみ | - |
| 完了済み | 完了のみ | - |

### 4.4 ToDoカード

| 要素 | 表示内容 |
|------|----------|
| タイトル | ToDoのtitle |
| 説明 | ToDoのdescription（あれば） |
| 作成日 | 「作成日: YYYY/MM/DD HH:MM」 |
| 完了ボタン | 「完了にする」or「未完了にする」 |
| 削除ボタン | 「削除」 |

### 4.5 完了状態の表示

| 状態 | 表示 |
|------|------|
| 未完了 | 通常表示 |
| 完了 | 背景グレー、取消線、透明度70% |

## 6. 操作フロー

### 6.1 ToDo追加フロー

```
1. タイトルを入力
2. 説明を入力（任意）
3. 追加ボタンをクリック or Enterキー
4. → フォームがクリア
5. → リストに新しいToDoが表示
6. → 統計情報が更新
```

### 6.2 完了切替フロー

```
1. ToDoの「完了にする」ボタンをクリック
2. → ToDoの見た目が変化（グレー、取消線）
3. → ボタンが「未完了にする」に変化
4. → 統計情報が更新
```

### 6.3 削除フロー

```
1. ToDoの「削除」ボタンをクリック
2. → 確認ダイアログ表示
3. OKをクリック
4. → リストからToDoが消える
5. → 統計情報が更新
```
