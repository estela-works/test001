## 4. リクエスト

### 4.1 パスパラメータ

| パラメータ | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| id | Long | 必須 | ToDoのID |

### 4.2 クエリパラメータ

なし

### 4.3 リクエストボディ

```json
{
  "title": "更新後タイトル",
  "description": "更新後説明",
  "completed": true,
  "assigneeId": 2
}
```

| フィールド | 型 | 必須 | 制約 | 説明 |
|-----------|-----|------|------|------|
| title | String | 必須 | 空文字不可 | タイトル |
| description | String | 任意 | - | 説明 |
| completed | boolean | 任意 | - | 完了フラグ |
| assigneeId | Long | 任意 | 存在するユーザーID | 担当者ID |

---

## 5. レスポンス

### 5.1 ステータスコード

| コード | 条件 |
|--------|------|
| 200 | 更新成功 |
| 400 | バリデーションエラー（担当者ID不正） |
| 404 | 指定IDが存在しない |

### 5.2 レスポンスボディ（成功時）

```json
{
  "id": 1,
  "title": "更新後タイトル",
  "description": "更新後説明",
  "completed": true,
  "createdAt": "2025-12-22T10:00:00",
  "assigneeId": 2,
  "assigneeName": "佐藤花子"
}
```

---

## 6. 処理フロー

```
1. リクエスト受信（パスパラメータ id、リクエストボディ取得）
   │
2. TodoMapper.findById(id) で存在確認
   │  └─ 存在しない → 404エラー
   │
3. バリデーション
   │  └─ assigneeId 指定時、UserMapper.findById(assigneeId)
   │       └─ 存在しない → 400エラー
   │
4. 既存Todoの値を更新
   │
5. TodoMapper.update(todo) 呼び出し
   │
6. レスポンス返却（200 + 更新後のTodo）
```
