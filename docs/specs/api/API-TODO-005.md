## 4. リクエスト

### 4.1 パスパラメータ

なし

### 4.2 クエリパラメータ

なし

### 4.3 リクエストボディ

```json
{
  "title": "タスク名",
  "description": "説明",
  "assigneeId": 1
}
```

| フィールド | 型 | 必須 | 制約 | 説明 |
|-----------|-----|------|------|------|
| title | String | 必須 | 空文字不可 | タイトル |
| description | String | 任意 | - | 説明 |
| assigneeId | Long | 任意 | 存在するユーザーID | 担当者ID |

---

## 5. レスポンス

### 5.1 ステータスコード

| コード | 条件 |
|--------|------|
| 201 | 作成成功 |
| 400 | バリデーションエラー（タイトル未入力、担当者ID不正） |

### 5.2 レスポンスボディ（成功時）

```json
{
  "id": 1,
  "title": "タスク名",
  "description": "説明",
  "completed": false,
  "createdAt": "2025-12-22T10:00:00",
  "assigneeId": 1,
  "assigneeName": "山田太郎"
}
```

---

## 6. 処理フロー

```
1. リクエスト受信（リクエストボディ取得）
   │
2. バリデーション
   │  ├─ title が空 → 400エラー
   │  └─ assigneeId 指定時、UserMapper.findById(assigneeId)
   │       └─ 存在しない → 400エラー
   │
3. Todoエンティティ生成
   │  ├─ completed = false
   │  └─ createdAt = 現在日時
   │
4. TodoMapper.insert(todo) 呼び出し
   │
5. レスポンス返却（201 + 作成されたTodo）
```

---

## 7. エラー処理

| ケース | HTTPステータス | エラーメッセージ |
|--------|---------------|-----------------|
| title が空または未指定 | 400 | Title is required |
| assigneeId が存在しないユーザー | 400 | Assignee not found |
