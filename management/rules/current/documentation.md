# ドキュメンテーションルール

## マネジメントチケットのドキュメント規約

### ticket.md の構成

```markdown
# チケット名

## 目的
なぜこのマネジメントタスクを実施するのか

## タスク
- [ ] タスク1
- [ ] タスク2
- [ ] タスク3

## ステータス
- **現在**: Open / In Progress / Done
- **開始日**: YYYY-MM-DD
- **完了日**: YYYY-MM-DD（完了時に記入）

## 関連情報
- 関連チケット
- 参照ドキュメント
- 外部リンク
```

### notes.md の運用

#### 作業中

- 日付ごとに作業ログを記録
- 気づき・メモを随時追記
- 決定事項を明記

#### 完了時

- 振り返りを追記
  - うまくいったこと
  - 改善すべきこと
  - 次回への引き継ぎ事項

### ルール定義ドキュメント

チケット内で `rules/` を使う場合:

```
rules/
├── workflow.md          # ワークフロー変更
├── documentation.md     # ドキュメントルール変更
└── development.md       # 開発プロセス変更
```

## グローバルルールとの統合

### チケット完了後の反映手順

1. チケット内ルールを確認
2. プロジェクト全体に適用すべきか判断
3. 適用する場合は `management/rules/current/` に反映
4. 古いルールは `management/rules/archive/` に移動（日付付きで保存）

### アーカイブ命名規則

```
archive/
├── YYYYMMDD_workflow.md
└── YYYYMMDD_documentation.md
```

## ドキュメントメンテナンス

- チケット完了時に関連ドキュメントを更新
- 変更履歴は Git コミットで管理
- 大きな変更は `management/tickets/` で管理
