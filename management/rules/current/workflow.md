# ワークフロールール

## マネジメントチケットの運用フロー

### チケットライフサイクル

```
チケット作成
    │
    ▼
management/tickets/open/ に格納
    │
    ▼
作業開始
    │
    ▼
作業完了
    │
    ▼
management/tickets/done/ に移動
```

## チケット作成時のルール

### フォルダ命名規則

```
YYYYMMDD_チケット名/
例: 20251225_ドキュメント体系見直し/
```

### 必須ファイル

- **ticket.md**: チケット情報（目的、タスク、ステータス）
- **notes.md**: 作業メモ・議事録

### オプショナルファイル

- **rules/**: このチケット専用のルール定義
  - チケットごとに異なる運用ルールを適用する場合に使用
  - チケット完了後、必要なルールは `management/rules/current/` に反映

## ルール変更のフロー

```
チケット作成
    │
    ▼
チケット内の rules/ でルールを定義
    │
    ▼
チケット作業中はそのルールを適用
    │
    ▼
チケット完了後
    │
    ├─→ プロジェクト全体に適用すべき場合
    │   └─→ management/rules/current/ に反映
    │
    └─→ チケット限定の場合
        └─→ チケット内 rules/ に保持したまま done/ に移動
```

## ステータス管理

### ticket.md のステータス表記

- **Open**: 未着手
- **In Progress**: 作業中
- **Done**: 完了

### フォルダ移動のタイミング

- **Done** になった時点で `management/tickets/done/` に移動
- 移動時に作業内容の振り返りを notes.md に記録

## マネジメントルールの優先順位

1. チケット内の `rules/`（最優先）
2. `management/rules/current/`（グローバルルール）
3. プロジェクトルート `CLAUDE.md`（基本ルール）
