# ルールアーカイブ

過去に使用されていたグローバルルールを保管するフォルダ。

## 目的

- ルール変更履歴の保存
- 過去の意思決定の参照
- ルール見直し時の比較材料

## ファイル命名規則

```
YYYYMMDD_ルールカテゴリ_変更概要.md
```

例:
- `20251225_workflow_チケット運用改善.md`
- `20251230_documentation_テンプレート追加.md`

## アーカイブ時の手順

1. `current/` から該当ルールファイルをコピー
2. 日付と変更概要を含むファイル名でリネーム
3. このフォルダに配置
4. `current/` のルールファイルを更新
5. Git にコミット
